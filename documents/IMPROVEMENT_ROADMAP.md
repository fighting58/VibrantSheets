# VibrantSheets 고도화 로드맵 (Improvement Roadmap)

VibrantSheets는 현재 기본적인 스프레드시트의 기능을 잘 갖추고 있으나, "프리미엄" 사용자 경험과 대규모 데이터 처리를 위해 다음과 같은 단계적 개선을 제안합니다.

---

## 1. 성능 및 확장성 개선 (Rendering & Scalability)

### 🚀 가상 스크롤링 (Virtual Scrolling / Windowing)
- **현재:** 모든 행과 열(예: 50x26)을 DOM에 직접 렌더링합니다. 행이 수천 개로 늘어나면 브라우저가 느려집니다.
- **개선:** `Canvas` 기반 렌더링 또는 `Virtual DOM` 기법을 적용하여 **현재 화면에 보이는 셀만 DOM에 유지**하도록 변경합니다.
- **기대 효과:** 수만 행 이상의 대규모 데이터에서도 끊김 없는 스크롤링과 빠른 반응성을 확보할 수 있습니다.

### 🎯 이벤트 위임 (Event Delegation)
- **현재:** 각 셀(`<td>`)마다 개별적으로 여러 개의 이벤트 리스너(mousedown, input, focus 등)를 연결하고 있습니다.
- **개선:** 그리드 컨테이너 혹은 테이블 레벨에서 단일 리스너를 두고, `event.target`을 통해 셀을 식별하는 방식으로 최적화합니다.
- **기대 효과:** 메모리 사용량을 줄이고 초기 렌더링 속도를 향상시킵니다.

---

## 2. 아키텍처 및 유지보수성 (Architecture & Maintainability)

### 📦 ES Modules (ESM) 전환
- **현재:** `<script>` 태그를 이용해 글로벌 스코프에서 스크립트를 순차적으로 로드합니다.
- **개선:** `import/export`를 사용하는 ES 모듈 구조로 전환하고, 필요 시 Vite와 같은 번들러를 도입합니다.
- **기대 효과:** 전역 오염 방지, 명확한 의존성 관리, 코드 스플리팅이 가능해집니다.

### 🏛️ 중앙 집중식 상태 관리 (State Management)
- **현재:** `VibrantSheets` 클래스 인스턴스 내부에 데이터와 UI 상태가 혼재되어 있습니다.
- **개선:** 데이터(Sheet Data)와 UI 상태(Selection, Active Sheet)를 분리하고, 상태 변경 시 렌더링이 트리거되는 구조(Store pattern)를 도입합니다.
- **기대 효과:** Undo/Redo 구현이 쉬워지고 버그 추적이 용이해집니다.

---

## 3. 사용자 경험 및 기능 고도화 (UX & Features)

### 🖱️ 컨텍스트 메뉴 (Context Menu)
- **개선:** 우클릭 시 복사, 붙여넣기, 행/열 삽입 및 삭제, 메모 추가 등의 기능을 제공하는 커스텀 메뉴를 추가합니다.

### 🧠 수식 입력기 고도화 (Advanced Formula Bar)
- **자동 완성:** 함수 이름 입력 시 제안 목록 표시 및 도움말 제공.
- **범위 선택:** 수식 입력 도중 그리드의 셀을 클릭하면 해당 참조(`A1`, `B2:C5` 등)가 자동으로 입력되는 기능.
- **구문 강조:** 수식 내 참조와 함수를 색상별로 구분하여 시각화.

### 🔄 Undo / Redo 시스템
- **개선:** `Command` 패턴을 활용하여 모든 편집 작업(값 변경, 스타일 적용, 행 삽입 등)에 대해 되돌리기 및 다시 실행 기능을 구현합니다.

---

## 4. 안정성 및 지속성 (Persistence & Reliability)

### 💾 자동 저장 및 복구 (Persistence)
- **개선:** `localStorage` 또는 `IndexedDB`를 활용하여 편집 중인 데이터를 주기적으로 저장합니다. 브라우저를 새로고침해도 마지막 상태로 복구됩니다.
- **최근 파일:** 이전에 열었던 파일 목록을 세션에 유지합니다.

---

## 5. 단계별 실행 계획 (Next Steps)

1.  **Short-term:** 자동 저장(Persistence) 및 세션 복구 시스템 구현.
2.  **Mid-term:** ES Modules 전환 및 프로젝트 구조 모듈화 강화.
3.  **Long-term:** 가상 스크롤링 엔진 도입 및 실행 취소(Undo/Redo) 시스템 구축.
