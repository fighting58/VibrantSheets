# Project Process

## 목적
이 문서는 VibrantSheets의 현재 구현 상태, 진행 단계, 다음 우선순위를 관리하기 위한 실행 문서입니다.

## 로드맵
1. Phase 10: 셀 병합
2. Phase 11: 셀 테두리
3. Phase 12: 인쇄 설정/미리보기
4. Phase 14: 이미지 삽입
5. Phase 13: Persistence & Recovery

## 현재 상태 (2026-03-28)

### 완료
- 기본 그리드/선택/편집/IME 입력 안정화
- 멀티 시트, 찾기/바꾸기, 수식 엔진(기본 함수/참조)
- 셀 서식(폰트/정렬/숫자형식) 및 병합
- 테두리 UI/적용/저장/불러오기 안정화
- 병합 셀 경계 테두리 우선순위 및 얇은선 렌더링 이슈 보정
- XLSX/CSV/VSHT import/export
- XLSX 라운드트립 개선
  - 병합 유지
  - strikethrough 유지
  - 배경/테두리 매핑 개선
  - `styles.xml` 복구 오류 유발 numFmt 버그 수정
- ExcelJS 로딩 실패 대응
  - drawing XML 음수 오프셋 클램프
  - drawing/vml 파트 제거 후 재시도
  - 사용자 안내 메시지 추가
- 통화 서식 보존 개선
  - KRW/USD 구분 파싱 및 재저장 시 통화 단위 유지
- 인쇄 기능(Phase 12 핵심)
  - 인쇄 모달/프리뷰
  - 인쇄 영역 기반 출력 흐름
  - 페이지 방향/헤더·푸터 슬롯 제어
  - 에디터 오버레이(선택 하이라이트 등) 인쇄 누출 최소화
  - 커스텀 페이지 생성으로 브라우저 인쇄 분할 보정
  - 1페이지 빈 출력 이슈 제거(인쇄 전용 DOM만 노출)
  - 인쇄 레이아웃 정밀도 개선(프리뷰/페이지 분할 정합성)
- 확장성 개선
  - 행 자동 확장(기존)
  - 열 자동 확장(붙여넣기/이동/입력 경로)
  - 컬럼 헤더 `AA`, `AB`... 표시 수정
- 기본 셀 크기 동기화(엑셀 기본값 기준)
  - 기본 열 너비/행 높이 기준 조정
  - 기존 문서 기본값 자동 마이그레이션
- 엑셀 스타일 텍스트 넘침 표시
  - 옆 셀이 비어있을 때만 overflow
  - 넘침 텍스트 클릭 시 실제 셀 선택

### 진행 중
- Phase 14 진행: 이미지 삽입 기능 고도화
  - XLSX 저장/불러오기 이미지 크기 보존 개선
  - 이미지 포함 파일 import 실패 케이스 추가 안정화

### 미완료
- Phase 14: 이미지 삽입/이동/리사이즈/저장
- Phase 13: localStorage 자동 저장/복구/최근 파일 목록

## 최근 결정 사항
- 병합 포함 붙여넣기는 내부 클립보드 메타데이터를 우선해 병합을 재적용한다.
- 인접 셀 테두리 충돌은 “나중에 설정한 테두리”가 우선되도록 맞춘다.
- 인쇄는 인쇄영역 기반을 기본 흐름으로 한다.

## 다음 단계 (우선순위)
1. 문서/체크리스트 기반 회귀 테스트 고정
2. Phase 14 구현 착수(이미지 삽입)
3. Phase 13 구현 착수(자동 저장/세션 복구)

## 모듈 참조
- `engines/vs_core.js`: UI/그리드/선택/편집/인쇄
- `engines/vs_find.js`: 찾기/바꾸기
- `engines/vs_io.js`: 파일 입출력(VSHT/XLSX/CSV)
- `engines/formula_engine.js`: 수식 계산
- `engines/bootstrap.js`: 초기화
