# VibrantSheets 프로젝트 통합 기획 및 상태 (Project Plan & Status)

## 1. 프로젝트 개요 (Overview)
VibrantSheets는 브라우저에서 동작하는 프리미엄 Excel 스타일 스프레드시트 앱입니다.
빠른 편집 성능, 직관적인 상호작용, 실사용 가능한 파일 호환성(.xlsx, .csv, .vsht)을 목표로 합니다.

## 2. 현재 상태 (Current Status)
**기준 일자:** 2026-03-28
**현재 단계:** Phase 14 진행 중 (이미지 삽입/저장/불러오기 안정화)

### 최근 반영 사항
- 기본 셀 크기 동기화 (엑셀 기본값 기준 64/22)
- XLSX 이미지 라운드트립 개선 (크기/좌표 보정 및 예외 처리 강화)
- 엑셀 스타일 텍스트 넘침 표시 고도화 및 보정
- 폰트 목록 정리 (깨진/중복 항목 제거)

### 진행 중
- 이미지 import/export 안정화 마감
- XLSX 파싱 실패 케이스(예: Malformed XML) 추가 점검 및 대응

### 완료된 주요 기능
- **편집 엔진:** 키보드 내비게이션, 범위 선택, 멀티 클립보드, Fill Handle(범위 자동 복사)
- **스타일/포맷팅:** 폰트/색상/정렬, Border 처리(우선순위/충돌 보정), 커스텀 포맷(Currency, Date, Percentage 등)
- **구조 제어:** 행/열 삽입 및 삭제, 시트 브라우징(추가/삭제/이름 변경), 자동 확장
- **함수 엔진:** SUM, CONCAT, LEFT, RIGHT 등 및 참조(A1, $A$1) 지원, 순환 참조 감지
- **파일 I/O:** `.vsht` (자체 포맷), `.xlsx` (ExcelJS 연동), `.csv`
- **인쇄 기능 (Phase 12):** 인쇄 모달/프리뷰, 페이지 분할 정합성, 인쇄 영역 기반 로직

## 3. 주요 정책 및 설계 결정 (Key Decisions)
- **클립보드 우선순위:** 병합 포함 붙여넣기 시 내부 클립보드 메타데이터를 시스템 클립보드 텍스트보다 우선 적용한다.
- **테두리 렌더링 충돌:** 인접 셀 간 테두리 스타일이 충돌할 경우 "나중에 설정한 테두리"가 시각적으로 우선되도록 처리한다.
- **인쇄 흐름:** 인쇄는 사용자가 지정한 '인쇄 영역(Print Area)'을 기준으로 하는 것을 기본 흐름으로 강제한다.

## 4. 단기 로드맵 (Short-Term Roadmap)
### Phase 14: 이미지 삽입 완료 및 안정화
- 로컬 파일 삽입, 셀 위 절대 배치(오버레이), Z-order 적용
- VSHT/XLSX 기반 이미지 메타데이터 및 base64 저장/불러오기
- 스크롤/줌/인쇄 환경에서의 이미지 좌표계 매핑

### Phase 13: Persistence & Recovery
- `localStorage` 또는 `IndexedDB` 기반 자동 저장 및 세션 복구 체계 구축
- 최근 작업한 파일 목록 유지
- 사용자 정의 리스트 (Fill Handle 용) 보존

## 5. 장기 고도화 로드맵 (Long-Term Improvement Roadmap)
### 5.1 성능 및 확장성 (Performance & Scalability)
- **가상 스크롤링 (Virtual Scrolling):** 화면 밖 셀 DOM 렌더링 제외로 대용량 시트 대응
- **이벤트 위임 고도화:** 각 셀 리스너 대신 그리드 컨테이너 단일 리스너 기반으로 메모리 최적화

### 5.2 아키텍처 (Architecture)
- **ES Modules 전환:** 구조 모듈화 및 향후 번들러 도입 준비
- **상태 관리 (State Management):** 데이터와 UI 상태 완벽 분리 및 Store 패턴 도입

### 5.3 사용자 경험 (UX Features)
- **우클릭 컨텍스트 메뉴:** 복사/붙여넣기, 삽입/삭제, 메모 등 빠른 접근
- **수식 입력기 고도화:** 자동 완성(Autocomplete), 구문 강조 방안 도입
- **Undo/Redo 시스템:** Command 패턴 인프라스트럭처 구축
